1. PROJECT OVERVIEW
================================================================================

TOVCH (also known as VYRU5 or Cursor Hold Game) is a real-time collaborative 
browser game for 2 players with virus battle mechanics.

**Core Experience:**
- Player 1 creates room → gets shareable ID
- Player 2 enters ID → joins same room
- Both players see each other's mouse cursors (red/blue followers)
- Players distribute 12 points across 12 virus parameters
- Both click READY → triggers virus battle (20x32 grid)
- Viruses spread every 500ms based on parameters
- Battle ends when one virus dominates (≥99%)

**Visual Style:**
- Retro neon aesthetics (Courier New / PIXY font)
- Neon colors: #00ff00 (green), #ff00ff (magenta), #00ffff (cyan)
- Dark background: #0f0f23 (dark blue)
- Grid overlay on canvas

5. KEY MECHANICS
================================================================================

VIRUS PARAMETERS (12 total, 12 points to distribute):
------------------------------------------------------
1. Aggression ⚔️ - Attack strength
2. Mutation 🧬 - Mutation chance
3. Speed ⚡ - Spread speed
4. Defense 🛡️ - Defense against capture
5. Reproduction 🦠 - Reproduction rate
6. Stealth 👻 - Stealth (not implemented)
7. Virulence ☣️ - Destruction power
8. Resilience 💪 - Survival
9. Mobility 🚶 - Movement
10. Intellect 🧠 - Strategy (not implemented)
11. Contagiousness 🫁 - Infection rate
12. Lethality 💀 - Lethality

BATTLE MECHANICS:
-----------------
- Grid: 20 columns × 32 rows = 640 cells
- Initial: Virus A (red) at top center, Virus B (blue) at bottom center
- Tick rate: 500ms (2 times per second)
- Spread chance: 30% to empty neighbor
- Conflict: 50% chance to capture opposing virus cell
- Win condition: ≥99% territory OR opponent eliminated
- Max ticks: 1000 (auto-end)

6. NETWORK COMMUNICATION
================================================================================

CLIENT → SERVER MESSAGES:
-------------------------
| Message Type         | Data                              | Purpose           |
|----------------------|-----------------------------------|-------------------|
| mflUpdate            | { isCreator, x, y }               | Mouse position    |
| updateObjectHover    | { objectId, isHovered }           | Orb hover         |
| startDragObject      | { objectId, startX, startY }      | Start drag        |
| updateObjectPosition | { objectId, x, y }                | Update position   |
| stopDragObject       | { objectId, endX, endY }          | Stop drag         |
| chatMessage          | { message, timestamp }            | Send chat         |
| updateVirusParams    | { params: { [key: string]: num }} | Update params     |
| toggleReady          | { isReady: boolean }              | Toggle ready      |
| startBattleNow       | {}                                | Start battle      |

SERVER → CLIENT MESSAGES:
-------------------------
| Message Type          | Data                              | Purpose          |
|-----------------------|-----------------------------------|------------------|
| mflUpdate             | { playerId, isCreator, x, y }     | Sync follower    |
| objectHoverChanged    | { objectId, isHovered, hoveredBy }| Sync hover       |
| objectDragStarted     | { objectId, playerId }            | Drag started     |
| objectPositionUpdated | { objectId, x, y }                | Position update  |
| objectDragStopped     | { objectId, playerId }            | Drag stopped     |
| chatMessage           | { playerId, playerName, msg, ts } | Chat message     |
| virusParamsUpdated    | { playerId, params }              | Params update    |
| playerReadyStatus     | { playerId, isReady }             | Ready status     |
| startCountdown        | { battleGrid, width, height }     | Start countdown  |
| virusTick             | { tick, battleGrid }              | Battle tick      |
| virusBattleEnded      | { winner, virusACount, virusBCount } | Battle ended |

- ☰ (left): Open controls sidebar
- ☰ (right): Open virus params sidebar

- JOIN ROOM: Join existing room (enter ID first)
- RANDOMIZE: Random virus params (12 points)
- READY: Ready for battle (disabled until 12 points distributed)

9. TROUBLESHOOTING
================================================================================

Common Issues:
1. Battle not starting
   - Check both players clicked READY
   - Verify 12 points distributed (READY button enabled)
   - Check console for "startCountdown" message
   - Verify NetworkManager subscribes to 'startCountdown'

2. Connection problems
   - Check server is running (port 2567)
   - Check client is running (port 3000)
   - Verify WebSocket connection in console

3. Battle grid not visible
   - Check BattleRenderer.initGrid() called
   - Verify grid dimensions match (20x32)
   - Check BattleRenderer.show() called

4. Console warnings "onMessage() not registered"
   - Add handlers for virusParamsUpdated, playerReadyStatus
   - These are non-critical (warnings only)

Development Tips:
- Server runs on port 2567 by default
- Client runs on port 3000 by default
- Check browser console for [BattleManager], [NetworkManager] logs
- Use /health endpoint to verify server
- Battle ticks every 500ms (check server logs)

10. CURRENT KNOWN ISSUES (v11)
================================================================================

1. Unregistered Message Types (Non-Critical)
   - virusParamsUpdated: Handler missing in NetworkManager
   - playerReadyStatus: Handler missing in NetworkManager
   - Impact: Console spam, but functionality works

2. Battle Renderer Performance
   - 640 cells × 60 FPS may cause lag on low-end devices
   - Consider reducing cell count or update rate

3. Params Not Used in Battle
   - Server spread logic ignores virus parameters
   - Currently: 30% spread, 50% conflict (hardcoded)
   - TODO: Implement param-based probabilities
