Ты — AI-ассистент. Твоя задача — создать полный текстовый снапшот проекта
для сохранения и передачи контекста.

================================================================================
ТРЕБОВАНИЯ К СНЭПШОТУ
================================================================================

1. СОЗДАЙ ПАПКУ
   - Название: debug1
   - Расположение: корень проекта

2. ФОРМАТ ИМЁН ФАЙЛОВ
   - Все файлы в ОДНОЙ плоской папке (БЕЗ подпапок)
   - Используй двойное подчёркивание __ как разделитель путей
   - Добавляй расширение .txt в конец

   Примеры:
   - client/src/main.ts → client__src__main.ts.txt
   - server/src/index.ts → server__src__index.ts.txt
   - package.json → package.json.txt
   - client/index.html → client__index.html.txt

3. КАКИЕ ФАЙЛЫ ВКЛЮЧИТЬ
   Включи ВСЕ важные файлы проекта:

   ✓ Root: package.json, README.md, *.bat скрипты, .md файлы
   ✓ Client: все .ts, .tsx, .html, .json, .css, .env, .npmrc
   ✓ Server: все .ts, .json файлы
   ✓ Типы: .d.ts файлы
   ✓ Конфиги: tsconfig.json, vite.config.ts, vitest.*

   НЕ включай:
   ✗ node_modules/
   ✗ dist/, build/
   ✗ .git/
   ✗ *.map файлы
   ✗ package-lock.json
   ✗ Временные файлы, логи

4. СОДЕРЖИМОЕ ФАЙЛОВ
   - Копируй полное содержимое каждого файла
   - Сохраняй кодировку UTF-8
   - Сохраняй все комментарии и документацию

5. СОЗДАЙ ИНДЕКСНЫЙ ФАЙЛ
   Создай файл INDEX.txt со списком всех файлов:

   ============================================================================
   TOVCH (VYRU5) - PROJECT SNAPSHOT INDEX
   ============================================================================
   Date: YYYY-MM-DD
   Version: vX

   FILE LIST (N files total):
   ----------
   1.  package.json.txt              - Root configuration
   2.  client__index.html.txt        - Main HTML template
   ...

   PROJECT STRUCTURE:
   ------------------
   [Дерево проекта]

   TECH STACK:
   -----------
   [Технологии]

   ============================================================================

6. СОЗДАЙ ФАЙЛ АРХИТЕКТУРЫ
   Создай файл ARCHITECTURE.txt с подробным описанием:

   - Обзор проекта
   - Структура проекта
   - Tech stack
   - Архитектурная диаграмма (ASCII)
   - Сетевая коммуникация
   - Как запускать
   - Ключевые особенности
   - Известные проблемы

================================================================================
АВТОМАТИЗАЦИЯ (PowerShell)
================================================================================

Для автоматического создания снапшота используй:

# Client файлы
Get-ChildItem -Path 'client' -Recurse -File -Exclude *.map |
  Where-Object { $_.DirectoryName -notmatch 'node_modules|dist|__tests__' } |
  ForEach-Object {
    $relativePath = $_.FullName.Replace($PWD.Path, '').Replace('\', '__')
    $content = Get-Content $_.FullName -Raw
    $destPath = 'debug1\' + $relativePath + '.txt'
    New-Item -ItemType Directory -Force -Path (Split-Path $destPath -Parent) | Out-Null
    Set-Content -Path $destPath -Value $content -Encoding UTF8
  }

# Server файлы
Get-ChildItem -Path 'server' -Recurse -File -Exclude *.map |
  Where-Object { $_.DirectoryName -notmatch 'node_modules|dist|__tests__' } |
  ForEach-Object {
    $relativePath = $_.FullName.Replace($PWD.Path, '').Replace('\', '__')
    $content = Get-Content $_.FullName -Raw
    $destPath = 'debug1\' + $relativePath + '.txt'
    New-Item -ItemType Directory -Force -Path (Split-Path $destPath -Parent) | Out-Null
    Set-Content -Path $destPath -Value $content -Encoding UTF8
  }

# Root файлы
Get-ChildItem -Path '.' -File |
  Where-Object { $_.Name -match '\.(json|md|txt|bat)$' -and $_.Name -notmatch 'package-lock' } |
  ForEach-Object {
    $content = Get-Content $_.FullName -Raw
    $destPath = 'debug1\' + $_.Name + '.txt'
    Set-Content -Path $destPath -Value $content -Encoding UTF8
  }
